# 需求文档 (PRD): 会话提问分析与进化引擎 (My Questions)

## 1. 背景与目标

随着用户与 LLM 对话量的增加，如何系统性地复盘自己的提问质量、识别思维漏洞并优化提问流程变得至关重要。本项目旨在提供一个本地化的工具，从海量历史会话中提取用户提问，并通过可视化和分治分析手段，辅助用户进行“提问能力进化”。

### 核心目标

- **集中化管理**：在一个视图中查看跨项目的所有连续提问。
- **质量度量**：基于现有 Coach 模块的评分，筛选出低质量提问。
- **LLM 适配**：提供适配 LLM 上下文限制的数据导出能力，支持分治分析法。
- **跨端调用**：支持浏览器 UI 访问与命令行（curl/wget）直接调用。

## 2. 用户场景

| 场景         | 描述                                                                                     |
| :----------- | :--------------------------------------------------------------------------------------- |
| **自我复盘** | 用户想看看最近一周在 `redis` 学习中，自己的提问是否越来越深刻，还是依然在原地打转。      |
| **模式识别** | 用户将评分低于 60 分的所有提问交给 LLM，让其分析自己在表达需求时是否存在逻辑断层。       |
| **流程优化** | 通过观察不同项目的提问统计，发现某个项目提问效率极低，进而反思该项目的预热知识是否不足。 |

## 3. 功能需求

### 3.1 提问提取与筛选

- **自动提取**：从所有会话文件中自动识别 `type: "user"` 的消息。
- **会话聚合**：以会话为单位进行聚合展示。
- **多维过滤**：
  - 按 **项目名称** 过滤。
  - 按 **表达质量分数**（Good > 80, Problematic < 80）过滤。
  - 支持关键词检索。

### 3.2 可视化展示

- **项目概览**：展示提问数最多、评分分布最广的核心项目卡片。
- **详情视图**：支持折叠展示会话内的完整提问链路。
- **快速跳转**：点击可直接进入原始会话详情页查看完整上下文（包含 AI 回复）。

### 3.3 数据导出 (LLM Bridge)

- **Markdown 适配**：一键将当前筛选后的结果转换为 Markdown 格式。
- **分阶段导出**：支持分页，避免一次性导出过多内容导致 LLM 内存溢出。
- **纯文本接口**：支持 `Accept: text/plain` 协议，方便开发者通过 shell 脚本自动化获取数据。

## 4. 非功能需求

- **性能**：支持 300+ 会话的秒级筛选与加载。
- **类型安全**：前后端严格共享 TypeScript 接口定义。
- **安全性**：数据纯本地存储，不上传至任何云端。
