# 使用指南: 如何利用 LLM 深度分析您的提问模式

本指南旨在指导您如何利用本项目提供的“My Questions”功能，结合分治分析法（Map-Reduce），让 LLM 帮助您识别提问中的逻辑缺陷、流程性问题并优化沟通效率。

---

## 核心流程：分治分析法 (Map-Reduce)

当您的提问记录非常庞大时，直接全部复制给 LLM 会导致上下文溢出。我们推荐以下三步走流程：

### 第一步：按项目提取 (Map)
不要尝试一次性分析所有项目。不同项目的提问逻辑（如源码学习 vs 业务开发）差异很大。

1.  进入 **My Questions** 页面。
2.  在顶部筛选器中选择一个**特定项目**（如 `redis`）。
3.  点击 **"Copy Current View"**。
4.  将内容粘贴给 LLM。

### 第二步：批次化总结 (Shuffling & Summary)
如果您在某个项目下有数十个会话，请利用分页或滚动加载：

1.  先复制前 10 个会话。
2.  **发送 Prompt 给 LLM**：
    > "以下是我在 [项目名] 中的前 10 组提问记录。请分析并识别出我在表达需求、逻辑连贯性、以及背景信息提供方面的 3 个主要问题。请用简练的列表给出。"
3.  再复制接下来的 10 个会话。
4.  **发送二次更新 Prompt**：
    > "这是接下来的 10 组提问记录。请结合你之前的分析，更新并完善我的提问缺陷列表。如果发现了新的重复模式请指出，如果之前的某个结论被强化了也请说明。"

### 第三步：综合诊断 (Reduce)
在完成特定项目或多个项目的分批分析后，让 LLM 给出最终的进化建议。

**发送最终 Prompt**：
> "基于你对以上所有批次的分析，请为我制定一个专属于我的『提问改进指南』。请侧重于：
> 1. 哪些流程性的操作是我经常忽略的（如：未提供环境背景）？
> 2. 我的提问路径是否存在发散、不聚焦的问题？
> 3. 我应该如何调整第一句提问的结构以获得最高效的回复？"

## 常用工具与接口

### 1. 命令行自动化 (推荐)
如果您习惯使用脚本，可以使用以下命令快速获取指定批次的 Markdown 数据，直接重定向到文件或管道：

```bash
# 获取 redis 项目中最新的 10 组提问
curl -H "Accept: text/plain" "http://localhost:3001/api/user-questions?project=redis&limit=10" > analysis_batch.md
```

### 2. UI 筛选器
*   **搜索框**：输入关键词（如 "bug", "how to"）来聚焦分析特定类型的提问。
*   **折叠/展开**：在复制前，快速浏览会话标题，通过点击右侧链接查看完整对话，确认该会话是否具有代表性。

---

## 注意事项
*   **忽略评分**：目前的评分系统尚在完善中，分析时请以“提问内容的逻辑性”和“LLM 的实际反馈质量”为准。
*   **数据隐私**：所有操作均在本地完成，您的提问记录不会被上传，请放心使用。
